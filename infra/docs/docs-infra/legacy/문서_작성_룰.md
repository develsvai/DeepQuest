# 문서 생성·수정 룰 (프롬프트)

인프라/docs-infra 및 k8s/docs 문서를 생성·수정할 때 AI와 사람이 따를 기준입니다.  
이 문서를 "문서 작성/수정 시 참고할 룰"로 프롬프트에 포함하거나, 수동 편집 시 체크리스트로 사용하세요.

**트래픽 테스트(Task) 문서:** `DeepQuest_Traffic_Test` 폴더 내 Task 공통 문서·결과 README는 위 룰에 더해 [DeepQuest_Traffic_Test/문서_작성_룰_트래픽테스트.md](DeepQuest_Traffic_Test/문서_작성_룰_트래픽테스트.md)를 따른다.

---

## 1. 메타데이터 (상단 고정)

- **위치**: 모든 문서 **맨 위**에만 둔다. 하단 메타데이터는 사용하지 않는다.
- **형식** (동일하게 통일):
  ```text
  **업데이트 날짜:** YYYY-MM-DD
  **작성자:** DeepQuest Team Haru2
  ```
- **포함하지 않을 항목**: 버전(version), 작성일(작성일 대신 업데이트 날짜만 사용).
- 메타데이터 다음에 구분선 `---` 를 넣은 뒤 제목(# )과 본문을 쓴다.

---

## 2. 이모지 사용

- **원칙**: 문서 본문·표·제목에서 **이모지는 사용하지 않는다** (📋 🎯 ✅ ❌ 🔥 등 전부 제거).
- 예외: 코드/설정 예시 안에 있는 문자열(예: 로그 메시지)은 필요 시만 유지.

---

## 3. 문서명·링크

- **문서명 변경 시**: 같은 폴더 또는 관련 문서 안의 **모든 내부 링크**를 새 문서명에 맞게 수정한다.
- **링크 형식**: 상대 경로 사용. 파일명에 공백이 있으면 URL 인코딩(`%20`) 적용.
  - 예: `[트러블슈팅](./트러블슈팅.md)`, `[K8S_배포 가이드](./K8S_배포%20가이드.md)`
- **깨진 링크**: 삭제·이름 변경된 문서를 가리키는 링크는 반드시 새 경로/이름으로 갱신한다.

---

## 4. 피드백 반영 시

- **표시**: 사용자 피드백이 `->` 또는 `<-` 로 문장/문단 옆에 붙어 있는 경우, **그 피드백만** 반영한다.
- **반영 후**: 피드백 문구 자체(예: "문장 다시쓰셈", "이것만 놔두셈")는 문서에서 **삭제**한다.
- **문장 재작성**: "이해하기 난해하다" / "뭔 말인지 모르겠다" 등이 있으면, 괄호·줄임을 줄이고 **한 문장씩 짧고 명확하게** 다시 쓴다.
- **추가 금지**: 사용자가 "~만 해둬", "~만 넣어"라고 한 것만 추가하고, **지시하지 않은 내용은 넣지 않는다.**

---

## 5. 목차·섹션 구조

- **목차**: 문서 상단에 목차(목차)를 두고, **앵커 링크가 실제 제목과 일치**하도록 한다.
  - 제목이 `## 3. Kubernetes 프로덕션 설정` 이면 링크는 `#3-kubernetes-프로덕션-설정` 형태로 맞춘다.
- **번호 체계**: 트러블슈팅·체크리스트 등은 **1, 2, 3, …** 처럼 끊기지 않고 이어지는 번호를 사용한다.
- **통합 시**: 여러 문서를 하나로 합칠 때(예: Jenkins 트러블슈팅 → 트러블슈팅) 기존 번호 체계에 이어서 붙인다(예: 13번 Jenkins 파이프라인).

---

## 6. 내용 이동·삭제

- **이동**: "A 문서 내용을 B 문서로 옮겨라"고 하면, B에 내용을 **추가**한 뒤 A에서는 해당 블록을 **삭제**한다.
- **삭제만 요청된 경우**: "조치는 쓰지 말고 ~만 해두셈" → **조치/해결 절차 문단은 쓰지 않고**, 요청한 문장(예: "DB 비밀번호에 특수문자 쓰지 말 것")만 남긴다.
- **중복 제거**: "다른 문서와 중복이니 삭제"라고 된 섹션은 해당 문서에서 제거하고, 필요 시 "자세한 내용은 [다른 문서](링크)" 한 줄로 대체한다.

---

## 7. 기술 문서(가이드·설계서)

- **설명 보강**: "이게 뭔 말이냐" / "이유를 적어라" 피드백이 있으면, **무슨 의미인지·왜 그래야 하는지** 한두 문장으로 명시한다.
  - 예: Clerk `pk_test_*` vs `pk_live_*` → "test는 개발용, live는 실제 사용자 인증용이므로 프로덕션에서는 live로 설정해야 함" 등.
- **코드/설정과 일치**: JSON·YAML·PromQL 등 **실제 파일/설정값**을 문서화할 때는, 복사·요약한 내용이 **원본과 일치**하도록 한다.
- **구분**: "현재 적용된 내용"과 "추가 권장/미적용"은 **섹션을 나누어** 구분해 쓴다(예: §1 현재 적용, §2 추가 권장).
- **경로·적용 방법**: 대시보드/스크립트 등은 **파일 경로**와 **적용 방법**(스크립트 이름, Import 절차 등)을 문서에 적어 둔다.
- **코드·YAML 주석**: 문서에 포함하는 YAML, 셸, 기타 코드 블록은 **각 라인**(또는 역할이 같은 라인 묶음)마다 그 라인/블록이 하는 역할을 주석으로 달 것. 읽는 사람이 복사·수정 시 의미를 바로 파악할 수 있도록 한다.
- **문서 사용법 안내 금지**: "한눈에 보기와 목차만 훑어도 ~", "이 문서를 읽으면 ~"처럼 **문서를 어떻게 읽으면 되는지**를 설명하는 문장은 문서 본문에 넣지 않는다. 그건 작성 시 요구사항(예: 한눈에 보게 써라)이므로 작성자가 따르면 되고, 독자가 알아야 할 내용이 아니다.

---

## 8. 날짜·확인 시점

- **문서 내 날짜**: "확인 시점", "사전 점검 결과" 등에는 **최신 확인일**을 사용한다(예: 2026-02).
- **업데이트 날짜**: 메타데이터의 업데이트 날짜는 **문서를 마지막으로 수정한 날**로 갱신한다.

---

## 9. 언어·용어

- **기본 언어**: 문서 본문은 **한국어**로 작성한다.
- **용어**: 기술 용어(PromQL, HPA, overlay, Kustomize 등)는 **원문 그대로** 두고, 필요 시 괄호로 한글 설명을 붙여도 된다.

---

## 10. 요약 체크리스트

문서를 만들거나 수정한 뒤 아래를 확인한다.

- [ ] 메타데이터가 **상단**에만 있고, 형식(업데이트 날짜, 작성자)이 통일되어 있는가?
- [ ] 이모지가 제거되어 있는가?
- [ ] 문서명 변경 시 **모든 내부 링크**가 새 이름/경로로 바뀌었는가?
- [ ] 사용자 피드백(`->` / `<-`)만 반영하고, 피드백 문구는 삭제했는가?
- [ ] 목차 앵커와 제목이 일치하는가? 번호가 끊기지 않는가?
- [ ] "조치 쓰지 말고 ~만" 요청 시 조치 문단 없이 요청 문장만 있는가?
- [ ] 코드/JSON/설정을 다룰 때 **현재 적용**과 **추가 권장**이 구분되어 있는가?
- [ ] 문서 내 YAML·코드 예시에 **각 라인(또는 역할 단위)별 역할 주석**이 달려 있는가?
- [ ] "이 문서를 어떻게 읽으면 되는지" 안내 문장이 본문에 들어가 있지 않은가?
- [ ] 업데이트 날짜와 문서 내 확인 시점이 최신으로 반영되어 있는가?

## 프롬프트로 붙여넣기용 (요약)

아래 문단을 문서 생성·수정 요청 시 함께 넣으면 된다.

```text
[문서 작성 룰]
- 메타데이터(업데이트 날짜, 작성자 DeepQuest Team Haru2)는 문서 상단에만 둔다. 버전·작성일 필드는 쓰지 않는다.
- 문서 전체에서 이모지는 사용하지 않는다.
- 문서명이 바뀌면 모든 내부 링크를 새 문서명으로 갱신한다. 파일명 공백은 %20으로 인코딩한다.
- 사용자가 -> / <- 로 남긴 피드백만 반영하고, 반영 후 피드백 문구는 삭제한다. 지시하지 않은 내용은 추가하지 않는다.
- 목차 앵커는 실제 제목과 맞추고, 섹션 번호는 1,2,3,… 으로 끊기지 않게 한다.
- "조치 쓰지 말고 ~만 해두라"면 조치 문단 없이 요청한 문장만 쓴다. 다른 문서로 내용 이동 시 원본에서는 해당 블록을 삭제한다.
- 기술 설명은 "무슨 의미/왜 필요한지"를 한두 문장으로 보강한다. 코드·JSON·설정은 원본과 일치하게 쓰고, "현재 적용"과 "추가 권장"을 구분한다.
- YAML·코드 블록은 각 라인(또는 역할 단위)마다 그 역할을 주석으로 표기한다.
- 문서 본문에 "이 문서를 어떻게 읽으면 되는지" 안내 문장(예: 한눈에 보기와 목차만 훑어도 파악할 수 있습니다)을 넣지 않는다. 독자용 내용만 쓴다.
- 기본 언어는 한국어, 기술 용어는 원문 유지. 업데이트 날짜와 문서 내 확인 시점은 최신으로 둔다.
```
