# 상태

날짜: `2026-03-28`
확인 시점: `2026-02-08 ~ 2026-03-28`
대상 범위: `deep-quest prod`, `배포 흐름`, `관측/부하 테스트`, `LangGraph/Redis`, `KEDA`, `DB/처리량 병목`

이 문서는 에이전트 도입 전 기존 인프라 문서와 저장소 이력을 바탕으로, 현재 운영 흐름에 직접 연결되는 내용만 기간 기준으로 정리한 예외 상태 문서다.

## 의도

- 현재 `status`와 별개로, `2026-02-08 ~ 2026-03-28` 구간에서 확인 가능한 인프라 구축 및 운영 흐름을 사실 기준으로 압축해 남긴다

## 현재 상태

- `2026-02-08` 기준 배포 표준은 이미 `Jenkins 빌드 -> Secret 반영 -> ArgoCD 수동 sync -> 배포 후 검증` 순서로 정리돼 있었고, 현재 `prod` 운영도 같은 큰 흐름 위에서 관리되고 있다
- `2026-02-13 ~ 2026-02-15`에는 K6 기준으로 Grafana 관측 체계와 web HPA 거동을 먼저 검증했고, baseline과 HPA 적용 후 결과를 분리해서 보는 운영 흐름이 잡혔다
- 그 다음 단계에서는 AI 런타임을 `langgraph build` 기반 프로덕션 이미지와 Redis/Postgres 연동 구조로 전환했고, session affinity, `N_JOBS_PER_WORKER`, AI service port `8000` 같은 운영 기준이 이 시기 문서와 저장소 변경으로 축적됐다
- 이후 Locust와 운영 로그 기준으로 `동시성 숫자 상향만으로 처리량이 늘지 않는다`는 점이 확인됐고, 실제 병목이 동기 그래프 구현, worker 점유, queue 적체, 세션 라우팅 쪽에 있음을 기준으로 후속 튜닝 흐름이 이어졌다
- `2026-03-24 ~ 2026-03-28`에는 `langgraph.run` metrics exporter, Redis 기반 Gemini 전역 rate limiter, backlog 기반 KEDA, Postgres connection 상향, 주요 AI 경로 async 전환, Jenkins 부분 빌드 태그 정합성 검증까지 현재 운영 문서 축으로 다시 정리됐다

## 알려진 이슈

- 이 문서는 과거 흐름 요약용이라, Jenkins UI 수동 변경, Harbor에만 남은 태그, cluster 내부 일회성 수동 조정처럼 저장소와 문서에 남지 않은 내용은 포함하지 못한다
- Postgres `max_connections = 500` 실측 메모의 실제 출처처럼 현재도 확정되지 않은 항목은 흐름에는 포함했지만, 최종 사실로 확정하지는 않았다
