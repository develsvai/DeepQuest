## 현재 상태

- `Jenkinsfile`, `k8s/overlays/prod/kustomization.yaml`, `k8s/argocd/application-prod.yaml` 대조 결과, prod 배포는 Jenkins가 `deploy` 브랜치의 prod overlay 이미지 태그를 갱신하고 ArgoCD가 이를 수동 sync로 반영하는 구조로 확인됐다
- 부분 빌드 시 교차 서비스 태그가 섞일 수 있던 Jenkins 동작은, 빌드하지 않은 이미지도 현재 `${DEPLOY_ENV}` 태그를 pull 후 새 `${BUILD_TAG}`로 재태깅해 push하고 prod overlay 세 이미지 태그를 모두 같은 `${BUILD_TAG}`로 갱신하도록 저장소 기준 수정됐다
- prod 배포 확인 포인트는 `docs/runbooks/prod-deployment.md`에, 부분 빌드 혼합 태그 증상과 대응 원칙은 `docs/troubleshooting/jenkins-and-build-agents.md`에 반영됐다
- Postgres 설정 경로 확인 결과 저장소 기준 `max_connections`는 `k8s/base/configmap.yaml`과 `docker/postgres/postgresql.conf` 모두 `200`이며, K8s StatefulSet은 ConfigMap 마운트를 제거한 상태라 실동작 설정 경로는 Postgres 이미지 내부 설정 파일 쪽으로 정리됐다
- 최신 `task`의 `진행` 기준으로 Jenkins 부분 빌드 수정 이후 재배포 결과를 확인했고, Harbor 재태깅과 `deploy` 브랜치 기준 태그 갱신은 현재 검증 범위에서 의도한 `${BUILD_TAG}` 기준으로 정상 동작하는 상태로 본다

## 알려진 이슈

- 과거 메모에 남아 있는 Postgres `max_connections = 500` 실측값은 현재 저장소 반영 경로가 확인되지 않아, 수동 변경인지 미반영 이미지인지 기록 오차인지 아직 확정되지 않았다
- `200ops` 목표를 위한 현재 병목 구간은 limiter, concurrency, queue backlog, DB connection, HPA, ingress 관점에서 아직 같은 시점 기준으로 다시 정리되지 않았다
- `nip.io` 기반 ingress 경로는 환경 제약에 따라 외부 경유 검증이 제한될 수 있다
- `pdf 분석 개선`은 현재 작업 범위에서 계속 보류 상태다
