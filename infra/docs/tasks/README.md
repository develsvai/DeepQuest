# 작업 허브

`tasks/`는 날짜별 작업 계획 문서를 쌓는 디렉터리다.

규칙:

- 작업 문서는 `task-YYYY-MM-DD.md` 형식으로 만든다
- 새 날짜에 `plan-task`를 실행하면 새 파일을 추가한다
- 하루 작업을 다 못 끝냈으면 그 파일에 진행 상태를 계속 남긴다
- 다음날 다시 이어갈 때는 새 날짜 파일을 만들고, 전날 미완료 사항을 이어서 적는다
- 현재 의도와 우선순위의 기준은 항상 `../status/` 아래 가장 최신 상태 파일이다
- 작업 파일은 항상 현재 인프라 문서 구조를 따르며 `계획`, `진행`, `보류 / 다음 액션`을 유지한다
- `plan-task`는 `계획`과 초기 `보류 / 다음 액션` 중심으로 쓴다
- `진행`은 실제 작업 후 `update-task`로 갱신한다
- `plan-task` 단계의 `진행`은 비워 두거나 시작 전 상태만 짧게 적는다
- 첫 `task`는 이전 작업 파일 없이도 최신 `status`만으로 생성할 수 있다
- task를 갱신할 때는 기존 인프라 task 문서의 섹션 순서와 톤을 우선 유지한다
- 하루 중 새로 확인된 현재 상태, 진행, 중단 지점은 `status`보다 `task`에 먼저 남긴다
- 다음날 `new-status`는 가장 최신 `status`와 가장 최신 `task`를 함께 읽고 오늘의 시작 기준 상태를 다시 만든다

권장 순서:

1. `../status/README.md`와 가장 최신 상태 파일을 읽는다
2. 첫 작업일이면 이전 task 없이 새 작업 파일 `task-YYYY-MM-DD.md`를 만든다
3. `plan-task`로 `계획`과 초기 `보류 / 다음 액션`을 기록한다
4. 실제 작업 후 `update-task`로 `진행`을 갱신한다
5. 다음 작업일 시작 시 `new-status`가 가장 최신 상태 파일과 작업 파일을 기준으로 새 상태 문서를 만든다
