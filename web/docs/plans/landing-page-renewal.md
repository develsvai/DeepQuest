# Deep Quest 랜딩 페이지 리뉴얼 기획

## 설계 철학: Start with Why

이 랜딩페이지는 Simon Sinek의 **"Start with Why"** (2009) 개념을 기반으로 설계되었다.

**Golden Circle 적용:**

```
WHY  → 왜 존재하는가? (목적, 신념)
HOW  → 어떻게 하는가? (차별화된 방법)
WHAT → 무엇을 하는가? (제품, 서비스)
```

**핵심 원칙:**

- **Why부터 시작**: 기능(What) 나열이 아닌, 서비스의 존재 이유와 신념을 먼저 전달
- **감정에 호소**: 사람들은 "무엇"을 사는 게 아니라 "왜"를 산다
- **공감 → 솔루션 순서**: 문제에 공감한 후 해결책 제시

---

## 1. Why - 서비스 존재 이유

> **"기술면접이 어려운 건 당신의 노력이 부족해서가 아닙니다.
> 경험을 깊이 있게 전달하는 방법을 만나지 못했을 뿐."**

### Why 핵심 요소

| 요소           | 내용                                     |
| -------------- | ---------------------------------------- |
| **Pain Point** | 기술면접에서 경험을 제대로 전달하지 못함 |
| **Reframe**    | 노력 부족 ❌ → 전달 방법의 부재 ✅       |
| **Promise**    | 경험을 깊이 있게 전달하는 방법 제공      |

---

## 2. 타겟 고객 정의

### Primary: 이직 준비 경력 개발자 (3-7년차)

- 경험은 충분하지만 정리가 안 됨
- "왜 그 기술을 선택했나요?"에 막힘
- 면접에서 경험의 깊이를 전달하지 못해 아쉬움

### Secondary: 취업 준비 주니어 개발자

- 사이드 프로젝트/부트캠프 경험 있음
- 경험이 적어서가 아니라, 있는 경험도 제대로 못 전달
- 기술 면접 질문에 대한 두려움

---

## 3. 페인 포인트 & 공감 시나리오

### 3.1 면접장에서 흔한 순간들

```
면접관: "프로젝트에서 가장 어려웠던 점이 뭐였나요?"
나: "어... 그게... 음... 여러 가지가 있었는데..."
(머릿속이 하얘지는 순간)
```

```
면접관: "왜 하필 그 기술을 선택했나요?"
나: "그게... 팀에서 원래 쓰던 거라서요..."
(분명 이유가 있었는데, 지금은 생각이 안 나는)
```

```
면접관: "트래픽이 10배가 되면 어떻게 하실 건가요?"
나: "...생각해본 적이 없어서요"
(이런 질문이 나올 줄 몰랐던)
```

### 3.2 면접 후의 막막함

```
"기술면접에서 떨어졌는데... 뭐가 부족했던 거지?"
(피드백도 없고, 뭘 개선해야 할지 모르는)
```

```
"분명 내가 만든 프로젝트인데, 왜 질문 받으면 말이 안 나오지?"
(머릿속에선 다 아는데, 입 밖으로 나오질 않는)
```

### 3.3 기존 대안들의 한계

```
"ChatGPT한테 면접 질문 뽑아달라고 했는데..."
→ "자기소개 해주세요", "지원 동기가 뭔가요?"
(이런 뻔한 질문 말고, 내 경험에 맞는 질문이 필요한데)
```

```
"면접 스터디 하려는데 믿을 만한 멤버 찾기가 너무 어려워"
(일정 맞추기도 힘들고, 피드백 퀄리티도 들쭉날쭉)
```

```
"현직자 멘토링은 1회에 5만원..."
(도움은 되겠지만, 여러 번 받기엔 부담되는)
```

### 3.4 핵심 인사이트

> **"경험은 분명히 있는데, 깊이 있게 설명하는 게 어려웠던 거예요"**

**그리고 기존 방법들은:**

- 피드백이 없거나 (혼자 준비)
- 질문이 뻔하거나 (일반 AI)
- 비용이 부담되거나 (멘토링)
- 일정/퀄리티가 불안정하거나 (스터디)

## 4. 섹션별 카피라이팅

### Section 1: Hero

#### 히어로 섹션 설계 기반

**1. 카피라이팅 핵심 전략**

| 원칙           | 적용 방법                                       |
| -------------- | ----------------------------------------------- |
| **3초의 법칙** | 헤드라인만으로 핵심 가치 전달                   |
| **고객 중심**  | 기능(AI 분석) ❌ → 이점(자신있게 답하기) ✅     |
| **명확/간결**  | 헤드라인 10단어 이내, 서브헤드 20단어 이내      |
| **후킹 요소**  | 문제점 언급 / 호기심 / 감정(두려움→자신감) 활용 |

**2. Why 핵심 메시지**

> "노력이 부족한 게 아니라, **전달하는 방법**을 만나지 못했을 뿐"

**3. 핵심 차별점 (USP)**

| 차별점                  | 고객 이점                | 왜 차별점인가                   |
| ----------------------- | ------------------------ | ------------------------------- |
| **STAR 기법 경험 분석** | 경험이 논리적으로 정리됨 | 검증된 답변 프레임워크로 체계화 |
| **깊이 있는 맞춤 질문** | 실제 면접 질문에 대비    | 내 경험 기반이라 뻔하지 않음    |
| **개인화된 피드백**     | 뭐가 부족한지 알 수 있음 | 혼자 준비할 때의 막막함 해소    |

---

#### 🏆 확정: 최종안 (Why 반영형)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  헤드라인:  "왜 그 기술을 선택했나요?"                         │
│                                                             │
│  서브헤드:  서류는 붙는데 면접에서 막힌다면,                   │
│            노력이 부족한 게 아닙니다. 방법을 만나지 못했을 뿐.  │
│                                                             │
│  ────────────────────────────────────────────────────────   │
│  이력서 업로드 → 경험 분석 → 맞춤 질문 → 피드백까지           │
│  ────────────────────────────────────────────────────────   │
│                                                             │
│           [ 내 경험으로 준비 시작하기 ]  ← Primary CTA        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**확정 이유:**

1. **타겟 상황 직접 언급**: "서류는 붙는데 면접에서 막힌다면" → 정확한 타겟팅
2. **Why 메시지 전달**: "노력이 부족한 게 아닙니다" → 자존감 보호 + 리프레이밍
3. **희망 제시**: "방법을 만나지 못했을 뿐" → 해결책 존재 암시
4. **전체 여정 표현**: 4단계 플로우로 서비스 가치 명확

---

### Section 2: Problem (공감 섹션) ✅ 확정

#### 구성: 2단계 공감 흐름

**Part 1 → Part 2 → 전환** 순서로 문제를 깊게 파고듦

---

#### Part 1: 면접장에서 막히는 순간

```
헤더: 기술면접, 이런 순간이 두렵지 않으셨나요?
```

| 면접관 질문                                 | 그 순간의 감정                 |
| ------------------------------------------- | ------------------------------ |
| "프로젝트에서 가장 어려웠던 점이요?"        | 머릿속이 하얘지는 순간         |
| "그 성능 개선, 구체적으로 어떻게 하셨어요?" | 대충 기억나는데 말이 안 나오는 |
| "트래픽이 10배면 어떻게 대응하실 건가요?"   | 생각해본 적 없는데...          |

---

#### Part 2: 기존 준비 방법의 한계

```
헤더: 그래서 어떻게 준비하셨나요?
```

| 방법         | 속마음                               |
| ------------ | ------------------------------------ |
| 혼자 연습    | "그래서 뭐가 부족했던 거지...?"      |
| ChatGPT 질문 | "이런 거 진짜 면접에서 물어보나...?" |
| 면접 스터디  | "일정도 안 맞고, 피드백도 애매해..." |
| 멘토링       | "좋긴 한데... 계속 받기엔 부담돼..." |

---

#### 전환 메시지

```
노력이 부족한 게 아닙니다.
전달하는 '방법'을 만나지 못했을 뿐.
```

---

#### 설계 의도

1. **Part 1**: 면접장 두려움 → "나도 저랬어" 공감
2. **Part 2**: 기존 대안 한계 → "맞아, 다 해봤는데 별로였어" 공감 심화
3. **전환**: Why 메시지로 연결 → "방법이 문제였구나" 인사이트
4. **흐름**: 문제 인식 → 기존 해결책 실패 → 새로운 해결책 필요성 자연스럽게 유도

---

### Section 3: Solution (How it works) ✅ 확정

#### 섹션 헤더

```
이제, 그 방법을 만나보세요
```

> Section 2 전환("전달하는 '방법'을 만나지 못했을 뿐")과 직접 연결

#### Step 1: 경험 분석

```
제목: 이력서를 업로드하세요
설명: AI가 당신의 경험을 회사별, 프로젝트별로 체계적으로 정리합니다.
     기술 스택, 역할, 기간이 자동으로 구조화됩니다.

이미지: 2.experience-result.png
```

#### Step 2: 핵심 성과 발굴

```
제목: 핵심 성과를 발굴합니다
설명: 각 경험에서 STAR 기반의 핵심 성과를 도출합니다.
     상황 → 행동 → 결과 → 교훈 구조로 정리되어
     면접에서 바로 활용할 수 있습니다.

이미지: 3.key-achievement-result.png
```

#### Step 3: 깊이 있는 질문 생성

```
제목: 면접관이 정말 궁금해하는 4가지 관점
설명: 실제 기술면접에서 면접관이 검증하는 핵심 관점을 기반으로
     당신의 경험에서 진짜 나올 수 있는 질문을 생성합니다.

이미지: 4-2.question-categories-detail.png
```

#### Step 4: 연습 & AI 피드백

```
제목: 답변하고, 피드백 받으세요
설명: 질문에 답변하면 AI가 강점과 개선점을 분석합니다.
     모범 답변과 비교하며 부족한 부분을 채워나가세요.

이미지: 6-2.question-feedback.png
```

### Section 6: CTA (최종 전환) ✅ 확정

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  헤드라인:  이제, 그 방법을 만나보세요                        │
│                                                             │
│  서브헤드:  기술면접, 더 이상 막히지 마세요.                  │
│                                                             │
│           [ 내 경험으로 준비 시작하기 ]  ← Primary CTA            │
│                 			                      │
│                                                             │
│           2분이면 첫 분석 결과를 받아볼 수 있어요                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**설계 의도:**

- **헤드라인**: Solution 헤더와 동일 ("이제, 그 방법을 만나보세요") — Why와 직접 연결
- **서브헤드**: 타겟 상황 + 결과 약속 ("더 이상 막히지 마세요")
- **Primary CTA**: Hero CTA와 동일 ("내 경험으로 준비 시작하기")
- **안심 메시지**: 시간 투자 부담 해소 (3분)

## 5. 톤 & 보이스 가이드

### 핵심 원칙

1. **공감 먼저, 솔루션 나중**: 고통을 인정하고 공감한 후 해결책 제시
2. **자존감 보호**: "부족하다"가 아닌 "방법을 만나지 못했을 뿐"
3. **구체적**: 추상적 약속 대신 실제 예시와 숫자
4. **겸손한 자신감**: 과장 없이 실제 가치 전달

### Do's and Don'ts

| Do                               | Don't                    |
| -------------------------------- | ------------------------ |
| "경험을 깊이 있게 전달하는 방법" | "완벽한 면접 준비"       |
| "이력서 기반 맞춤 질문"          | "AI가 알아서 해줍니다"   |
| "P99 800ms → 150ms로 개선"       | "성능을 획기적으로 개선" |
| "연습하고 피드백 받으세요"       | "합격을 보장합니다"      |

### 금지 표현

- ❌ "완벽한", "확실한", "보장"
- ❌ "당신은 부족해요"
- ❌ "이것만 하면 합격"
- ❌ 과장된 성공 사례
