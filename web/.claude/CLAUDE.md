# Web Project Local Rules

## 코드 탐색 도구 선택 가이드

> **TypeScript LSP 서버가 활성화되어 있으며 즉시 사용 가능합니다.**

코드 탐색 시 **작업 목적에 따라 적절한 도구를 선택**합니다. LSP 도구와 텍스트 기반 도구는 각각 강점이 다르므로, 상황에 맞게 사용해야 합니다.

### 도구 선택 기준

| 목적 | 적합한 도구 | 이유 |
|------|-------------|------|
| 심볼의 정확한 참조/정의 추적 | LSP | 컴파일러 수준의 의미론적 분석 |
| 문자열, 주석, 설정값 검색 | Grep | LSP는 비코드 요소를 추적하지 않음 |
| 파일 구조/패턴 탐색 | Glob | 빠른 파일 시스템 검색 |
| 타입 정보 확인 | LSP | 정확한 타입 추론 결과 제공 |
| 비코드 파일 (JSON, MD, 설정) | Read/Grep | LSP 지원 범위 밖 |

### LSP 도구 (의미론적 분석)

코드의 **구조와 의미**를 이해해야 할 때 사용합니다.

| 도구 | 사용 상황 |
|------|-----------|
| `LSP findReferences` | 함수/변수가 실제로 사용되는 모든 위치 |
| `LSP goToDefinition` | 심볼이 정의된 위치로 이동 |
| `LSP hover` | 타입 정보, 문서 주석 확인 |
| `LSP documentSymbol` | 파일 내 모든 심볼(함수, 클래스, 변수) 목록 |
| `LSP workspaceSymbol` | 프로젝트 전체에서 심볼 이름으로 검색 |
| `LSP goToImplementation` | 인터페이스/추상 클래스의 구현체 찾기 |
| `LSP prepareCallHierarchy` | 호출 계층 분석 준비 |
| `LSP incomingCalls` | 이 함수를 호출하는 곳들 |
| `LSP outgoingCalls` | 이 함수가 호출하는 곳들 |

### 텍스트 기반 도구 (패턴 검색)

**문자열 패턴**이나 **비코드 파일**을 다룰 때 사용합니다.

| 도구 | 사용 상황 |
|------|-----------|
| `Grep` | 문자열 리터럴, 주석, 정규식 패턴 검색 |
| `Glob` | 파일명/경로 패턴으로 파일 찾기 |
| `Read` | 파일 내용 직접 읽기, 설정 파일 확인 |

### 실제 상황별 도구 선택 예시

| 작업 | 잘못된 선택 | 올바른 선택 |
|------|-------------|-------------|
| "이 함수를 호출하는 곳 찾기" | Grep (문자열만 매칭) | `LSP findReferences` |
| "TODO 주석 전부 찾기" | LSP (주석 무시) | `Grep "TODO"` |
| "이 변수의 타입이 뭐지?" | Read (추론 불가) | `LSP hover` |
| "환경변수 사용처 찾기" | LSP (문자열 리터럴) | `Grep "process.env"` |
| "컴포넌트 파일들 찾기" | Grep (비효율적) | `Glob "**/*.tsx"` |
| "API 엔드포인트 URL 찾기" | LSP (문자열 값) | `Grep "/api/"` |
