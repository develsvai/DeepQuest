# i18n Rules (next-intl)

## Core Principle

All user-facing text MUST use next-intl. No hardcoded strings or conditional locale checks.

## File Organization

- Translation files are managed **per page** (one JSON per route)
- Files in `/locales/{ko,en}/` must match route structure
- Example: `src/app/[locale]/(protected)/dashboard/page.tsx` → `locales/{ko,en}/dashboard.json`
- **Shared namespaces** (`common.json`, `errors.json`): cross-page reusable keys (buttons, status, messages)
- **Adding new namespace requires updating both:**
  1. `src/i18n/request.ts` - runtime loading
  2. `src/types/i18n.d.ts` - TypeScript types

## NEVER Do This

```tsx
// ❌ Hardcoded text
<span>Submit</span>

// ❌ Conditional locale checks
{locale === 'ko' ? '제출' : 'Submit'}

// ❌ Object literals for translations
const labels = { ko: '제출', en: 'Submit' }
```

## ICU Message Format

### Pluralization (MUST use for counts)

```json
{ "items": "{count, plural, =0 {No items} one {# item} other {# items}}" }
```

### Select (MUST use for conditional text)

```json
{ "status": "{status, select, online {Online} offline {Offline} other {Unknown}}" }
```

### Date/Number in Messages

```json
{
  "memberSince": "Member since {date, date, short}",
  "price": "Price: {amount, number, currency}"
}
```

## Implementation Patterns

### Dynamic Values

```tsx
t('duration', { years: 2, months: 3 }) // "{years}년 {months}개월"
```

### Rich Text (`t.rich`) - Preferred

```tsx
t.rich('terms', {
  link: (chunks) => <Link href="/terms">{chunks}</Link>
})
```

### HTML Markup (`t.markup`) - Use with caution

```tsx
// ⚠️ Only for trusted, developer-controlled content
// NEVER use with user-generated content
t.markup('msg', { b: (chunks) => `<b>${chunks}</b>` })
```

### Optional Keys (`t.has`)

```tsx
if (t.has('optionalBio')) return <p>{t('optionalBio')}</p>
```

### Date/Number Formatting

```tsx
const format = useFormatter()
format.dateTime(new Date(), 'short')
format.number(1234.56, 'currency')
```

## Required Configurations

### Error Handling (`i18n/request.ts`)

```typescript
// MUST configure onError and getMessageFallback
onError(error) {
  if (error.code === IntlErrorCode.MISSING_MESSAGE) console.error(error)
  else reportToSentry(error)
},
getMessageFallback({ namespace, key }) {
  return `[Missing: ${namespace}.${key}]`
}
```

### TypeScript Type Safety (`types/i18n.d.ts`)

```typescript
// MUST declare for type-safe translations
declare module 'next-intl' {
  interface AppConfig {
    Messages: typeof import('@/locales/en/common.json')
    Locale: 'ko' | 'en'
  }
}
```

## Common Keys (Reuse these)

```
common.buttons.{submit|cancel|save|delete|edit|back|next}
common.status.{completed|pending|inProgress|failed}
common.messages.{loading|error|success}
```

## Exceptions

Acceptable hardcoded text:
- `console.log` messages (dev only)
- Developer-facing error messages
- HTML meta tags handled by next-intl
