# App Document Writing Rules

## 목적

루트 애플리케이션 문서를 갱신할 때 형식과 사실 기준을 통일한다.

## 규칙

- 상단에 작업 기준 날짜나 확인 시점을 필요에 맞게 명시한다
- 추정보다 실제 코드, 설정, 검증된 작업 결과를 우선한다
- 실행 절차와 배경 설명을 섞지 않는다
- 트러블슈팅은 재발 가능성이 있고 진단 가치가 있을 때만 분리한다
- 링크, 파일 경로, 명령은 실제 경로 기준으로 유지한다

## 문서 유형별 분리

- 상태 기록: `docs/status/`
- 작업 계획과 진행 기록: `docs/tasks/`
- 비교표/배경 문서: `docs/reference/`
- 장애 문서: `docs/troubleshooting/`
- 과거 원본: `lagacy/`, `docs-lagacy/`

## 원칙

- 루트 문서는 애플리케이션 작업 기준으로 유지한다
- 인프라 운영 절차나 운영 상태 스냅샷은 `infra/` 아래로 보낸다
- 같은 정보를 여러 문서에 반복 복제하지 않고, 기준 문서를 정해 갱신한다
