# 상태 허브

`status/`는 날짜별 상태 문서를 쌓는 디렉터리다.

규칙:

- 상태 문서는 `status-YYYY-MM-DD.md` 형식으로 만든다
- 하루를 시작할 때 `new-status`를 실행해 새 상태 파일을 만든다
- 같은 날짜 파일이 이미 있으면 그 파일을 계속 갱신한다
- 다른 skill이 `status`를 읽는다고 하면 항상 가장 최신 날짜 파일을 기준으로 본다
- 현재 의도, 진행 상황, 다음 액션의 기준은 항상 최신 상태 파일이다
- 상태 파일은 현재 인프라 문서 구조를 따르며 최소한 `의도`, `현재 상태`, `알려진 이슈`, `다음 작업`을 유지한다
- 새 상태 문서를 만들 때도 기존 상태 문서의 톤과 섹션 구조를 우선 맞춘다
- 새 작업일의 `status`는 전날 `status`와 가장 최신 `task`를 바탕으로 오늘 시작 시점의 `현재 상태`, `알려진 이슈`, `다음 작업`을 다시 정리한다
- 하루 시작 기준으로 `현재 상태`를 한 번 정리한 뒤에는 당일 진행 때문에 계속 덮어쓰지 않는다
- 당일 진행 상황과 실행 기록은 `status`보다 `task`에 남긴다
- 따라서 기본 흐름에서는 `update-status`를 사용하지 않는다

권장 순서:

1. 가장 최신 `status-YYYY-MM-DD.md`를 읽는다
2. 새 작업일이면 `new-status`로 오늘 파일을 만든다
3. `plan-task`는 최신 상태 파일을 기준으로 오늘 작업 파일의 `계획` 초안을 만든다
4. 실제 작업 기록은 `update-task`로 갱신한다
5. 진행 중 판단, 중단 지점, 실제 실행 결과는 최신 작업 파일에 계속 반영한다
6. 다음 작업일 시작 시 `new-status`가 전날 `status`와 최신 `task`를 기준으로 새 상태 파일을 만든다
