# 런북

반복 실행 절차와 운영 가이드를 둔다.

문서를 추가하거나 수정할 때는 먼저 `../reference/document-writing-rules.md`를 따른다.

## 사용 원칙

- 재현 가능한 절차만 남긴다
- 실제 경로와 명령을 유지한다
- 상단에 `확인 시점`과 `대상 환경`을 넣는다
- 섹션은 가능하면 `목적 -> 사전 확인 사항 -> 실행 순서 -> 검증 방법 -> 주의 사항` 흐름을 따른다
- 장애 원인 분석 문구는 `../troubleshooting/`로 보낸다
- 비교표나 배경 설명은 `../reference/`로 보낸다
- 상태 스냅샷은 `../infra_state/`에 남긴다
