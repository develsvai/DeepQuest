# 서버 레이어 체크리스트

## Router가 해야 할 일

- 입력 검증
- 인증/권한 체크
- service 호출
- 에러를 API 응답 형태로 변환

## Router가 하면 안 되는 일

- 긴 비즈니스 로직
- 직접 Prisma query 남발
- 외부 API 조합 로직
- 화면 전용 데이터 가공을 서비스 규칙처럼 고정

## Service가 해야 할 일

- 비즈니스 규칙
- 도메인 단위 query 조합
- transaction 경계
- 재사용 가능한 처리 흐름

## 점검 질문

- 이 로직이 다른 router에서도 재사용될 수 있으면 service로 가야 하지 않는가?
- router 파일이 입력/출력보다 도메인 규칙 설명으로 길어지지 않았는가?
- 페이지 경로 기준 이름이 service 구조를 오염시키지 않았는가?
