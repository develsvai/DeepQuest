# LangGraph Stream 체크리스트

## 최소 확인 항목

- `assistantId`가 현재 서버와 맞는가?
- `threadId`를 어디에 저장하고 복원하는가?
- 첫 진입과 재진입 흐름이 같은가?
- 로딩 중단 버튼이나 재시도 경로가 있는가?
- 에러가 사용자에게 보이는가?
- 완료 후 화면 상태가 서버 결과와 동기화되는가?

## 구현 포인트

- 단순 화면이면:
  로딩, 에러, 완료 상태만 먼저 맞춘다.
- 대화형 화면이면:
  `threadId` 유지와 메시지 append 순서를 먼저 맞춘다.
- 장기 실행 화면이면:
  새로고침 후 복원과 reconnect 전략을 먼저 정한다.
- 승인/확인 흐름이 있으면:
  interrupt 상태를 별도 UI 상태로 다룬다.

## 자주 깨지는 지점

- `threadId`를 잃어서 매번 새 대화가 생성됨
- 낙관적 메시지가 최종 서버 상태와 중복됨
- 로딩 상태만 있고 중단/재시도가 없음
- 서버 계약이 바뀌었는데 `messagesKey`나 렌더링 가정을 그대로 둠
