# Web 성능 체크리스트

## 1. Waterfall

- 서로 독립인 fetch가 직렬로 기다리지 않는가?
- 서버에서 미리 시작할 수 있는 비동기 작업을 너무 늦게 시작하지 않는가?

## 2. Server / Client 경계

- 정말 필요한 곳만 `use client`를 쓰는가?
- server component에서 끝낼 수 있는 데이터 준비를 client로 밀어내지 않았는가?

## 3. Rerender

- 큰 상태를 상위에서 한꺼번에 들고 있어 자식 전체가 자주 다시 그려지지 않는가?
- callback에서만 쓰는 값을 렌더 구독으로 묶지 않았는가?

## 4. Bundle / Hydration

- 무거운 라이브러리를 첫 진입에 함께 싣지 않는가?
- 조건부로만 필요한 UI를 정적 import로 고정하지 않았는가?

## 5. 렌더링 비용

- 긴 리스트나 복잡한 카드가 불필요하게 한 번에 모두 렌더되지 않는가?
- 정적인 JSX나 계산 결과를 매 렌더마다 새로 만들지 않는가?

## 6. 완료 전 질문

- 이 변경으로 `client` 영역이 전보다 커졌는가?
- fetch 시작 시점이 전보다 늦어졌는가?
- `pnpm check-all` 외에 체감 성능 저하를 확인해야 할 화면이 있는가?
