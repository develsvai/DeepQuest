# Gemini 프롬프트 체크리스트

## 기본 구조

형식이 중요한 프롬프트는 아래 순서를 기본으로 둔다.

1. 역할
2. 사용할 입력
3. 반드시 지켜야 할 제약
4. 출력 형식
5. 필요한 경우 짧은 예시

예시:

```text
역할:
- 너는 기술 면접 질문 생성기다.

입력:
- 지원자 경력 요약
- 채용 공고 요약

제약:
- 한국어로만 작성한다.
- 질문은 3개만 만든다.
- 각 질문은 2문장 이내다.
- 제공된 입력에 없는 사실은 추가하지 않는다.

출력 형식:
- JSON 배열
- 각 항목은 `question`, `intent`, `difficulty`
```

## 품질 점검 질문

- 이 프롬프트의 출력 계약이 후속 파서와 정확히 맞는가?
- 모델이 사용할 수 있는 입력 범위가 충분히 명시됐는가?
- 길이, 언어, 개수, 필드 같은 하드 제약이 빠지지 않았는가?
- 예시는 형식 안정화에 실제로 필요할 때만 넣었는가?
- "좋게 써줘" 같은 추상 표현 대신 판정 가능한 기준으로 바꿨는가?

## 흔한 수정 패턴

- 형식이 흔들릴 때:
  출력 형식을 bullet 대신 JSON, XML, 고정 섹션처럼 더 강한 구조로 바꾼다.
- 내용이 과장될 때:
  "제공된 입력만 사용"과 "없는 정보는 비워 둠"을 명시한다.
- 답이 장황할 때:
  항목 수, 문장 수, 글자 수 중 하나를 명시한다.
- 항목명이 자꾸 바뀔 때:
  허용 필드 목록과 순서를 고정한다.
