# Serena MCP 활용 가이드

## 개요

Serena MCP는 Claude Code의 코드 이해 및 편집 능력을 획기적으로 향상시키는 도구입니다. 단순 텍스트 기반 검색/치환이 아닌 **심볼 레벨**의 의미론적 코드 분석과 편집을 가능하게 합니다.

## 핵심 개념

### 심볼 기반 코드 이해
- 코드를 텍스트가 아닌 **구조화된 심볼**(클래스, 함수, 변수)로 이해
- Language Server Protocol(LSP) 활용으로 정확한 코드 관계 파악
- 토큰 사용량 최적화: 필요한 심볼만 선택적으로 읽기

### 효율적인 코드 탐색 전략
```
1. 전체 파일 읽기 ❌ → 심볼 개요 확인 ✅
2. 텍스트 검색 ❌ → 심볼 검색 ✅
3. 수동 참조 찾기 ❌ → 자동 참조 분석 ✅
```

## 주요 도구 및 사용법

### 1. 프로젝트 탐색 도구

#### `get_symbols_overview`
**용도**: 파일의 전체 구조를 빠르게 파악
```
# 사용 예시
- 새로운 파일을 이해해야 할 때
- 클래스의 메서드 목록을 확인할 때
- 파일의 전체 구조를 파악할 때
```

#### `find_symbol`
**용도**: 특정 심볼을 정확하게 찾기
```
# 매개변수
- name_path: "ClassName/methodName" (심볼 경로)
- include_body: true (코드 본문 포함 여부)
- depth: 1 (하위 심볼 탐색 깊이)

# 활용 패턴
- "User" → 모든 User 관련 심볼
- "User/__init__" → User 클래스의 생성자
- "/api/auth" → 최상위 api 모듈의 auth 함수
```

#### `find_referencing_symbols`
**용도**: 특정 심볼이 사용되는 모든 위치 찾기
```
# 리팩토링 시 필수
- 함수명 변경 시 모든 호출 위치 파악
- 클래스 수정 시 영향 범위 분석
- 의존성 체인 추적
```

### 2. 코드 검색 도구

#### `search_for_pattern`
**용도**: 정규 표현식으로 코드 패턴 검색
```
# 사용 시나리오
- 특정 패턴의 보안 취약점 검색
- 코딩 스타일 위반 사항 찾기
- 복잡한 코드 패턴 매칭

# 매개변수
- substring_pattern: 정규 표현식
- restrict_search_to_code_files: true (코드 파일만)
- context_lines_before/after: 3 (컨텍스트 라인 수)
```

### 3. 코드 편집 도구

#### `replace_symbol_body`
**용도**: 심볼 전체를 새로운 구현으로 교체
```
# 최적 사용 케이스
- 메서드 전체 리팩토링
- 클래스 구현 변경
- 함수 로직 완전 재작성
```

#### `insert_before_symbol` / `insert_after_symbol`
**용도**: 심볼 앞/뒤에 코드 삽입
```
# 활용 예시
- import 문 추가 (첫 심볼 앞에)
- 새 메서드 추가 (클래스 내 마지막 메서드 뒤에)
- 헬퍼 함수 추가
```

#### `replace_regex`
**용도**: 정규식 기반 정밀 수정
```
# 사용 원칙
- 작은 변경: 심볼 내 몇 줄만 수정
- 다중 교체: allow_multiple_occurrences 활용
- 와일드카드 적극 활용: .*? 으로 중간 부분 매칭
```

### 4. 메모리 관리 도구

#### `write_memory` / `read_memory`
**용도**: 프로젝트 컨텍스트 저장 및 활용
```
# 저장할 정보
- 프로젝트 아키텍처 개요
- 주요 디자인 패턴
- 코딩 컨벤션
- 자주 사용하는 유틸리티 위치
```

## 실전 활용 시나리오

### 시나리오 1: 대규모 리팩토링
```markdown
1. find_symbol로 대상 심볼 찾기
2. find_referencing_symbols로 모든 참조 파악
3. replace_symbol_body로 구현 변경
4. replace_regex로 참조 업데이트
5. 테스트 실행으로 검증
```

### 시나리오 2: 새 기능 추가
```markdown
1. get_symbols_overview로 파일 구조 파악
2. 유사한 기능 find_symbol로 분석
3. insert_after_symbol로 새 코드 추가
4. find_referencing_symbols로 통합 지점 확인
```

### 시나리오 3: 버그 수정
```markdown
1. search_for_pattern으로 문제 패턴 검색
2. find_symbol로 정확한 위치 확인
3. replace_regex로 정밀 수정
4. find_referencing_symbols로 영향 범위 확인
```

## 효율 극대화 팁

### ✅ DO (권장 사항)

1. **심볼 우선 접근**
   - 항상 get_symbols_overview로 시작
   - 전체 파일 읽기는 최후의 수단

2. **점진적 탐색**
   - depth=0으로 개요 파악 → depth=1로 상세 확인
   - include_body=false로 구조 파악 → true로 구현 확인

3. **정규식 활용**
   - 와일드카드(.*?)로 토큰 절약
   - 고유한 패턴으로 정확한 매칭

4. **메모리 활용**
   - 프로젝트별 컨벤션 저장
   - 반복 작업 패턴 기록

### ❌ DON'T (피해야 할 사항)

1. **비효율적 접근**
   - 전체 파일을 먼저 읽기
   - 심볼 도구 없이 텍스트 검색만 사용
   - 같은 내용을 여러 도구로 중복 읽기

2. **부정확한 수정**
   - 심볼 참조 확인 없이 이름 변경
   - 정규식 테스트 없이 대량 치환
   - 컨텍스트 무시한 기계적 수정

## 도구 선택 가이드

| 작업 유형 | 추천 도구 | 이유 |
|---------|---------|------|
| 새 파일 이해 | get_symbols_overview | 전체 구조 빠른 파악 |
| 함수 찾기 | find_symbol | 정확한 심볼 위치 |
| 리팩토링 | find_referencing_symbols | 영향 범위 파악 |
| 패턴 검색 | search_for_pattern | 유연한 검색 |
| 전체 교체 | replace_symbol_body | 심볼 단위 교체 |
| 부분 수정 | replace_regex | 정밀한 수정 |
| 코드 추가 | insert_before/after_symbol | 정확한 위치 삽입 |

## 성능 최적화 전략

### 토큰 사용량 절감
1. 필요한 심볼만 선택적으로 읽기
2. include_body는 필요시에만 true
3. 정규식 와일드카드로 중복 제거
4. 메모리 활용으로 반복 읽기 방지

### 작업 속도 향상
1. 병렬 심볼 검색 활용
2. 캐싱된 메모리 정보 재사용
3. 점진적 탐색으로 불필요한 읽기 방지

## 주의사항

- **문서 작업 시**: Serena 도구보다 일반 파일 도구 사용
- **코드 편집 시**: Serena 도구 우선 사용
- **대량 수정 시**: 항상 참조 확인 후 진행
- **복잡한 리팩토링**: 단계별 검증 필수
