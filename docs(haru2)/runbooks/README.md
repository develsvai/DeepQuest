# App Runbooks

루트 애플리케이션 작업 영역에서 반복 실행하는 절차 문서는 이 디렉터리에 둔다.

문서를 추가하거나 수정할 때는 먼저 `../reference/document-writing-rules.md`를 따른다.

## 사용 원칙

- 재현 가능한 절차만 남긴다
- Markdown 링크 대상은 작성 중인 문서 위치 기준 상대경로로 쓴다
- 명령과 설명 속 파일 경로는 실행 위치가 명확하게 드러나게 쓴다
- 상단에 `확인 시점`과 `대상 환경`을 넣는다
- 섹션은 가능하면 `목적 -> 사전 확인 사항 -> 실행 순서 -> 검증 방법 -> 주의 사항` 흐름을 따른다
- 장애 원인 분석 문구는 `../troubleshooting/`로 보낸다
- 비교표나 배경 설명은 `../reference/`로 보낸다
- 상태 기록과 진행 기록은 `../status/`, `../tasks/`에 남긴다
- 하나의 runbook은 하나의 반복 가능한 애플리케이션 작업 목표만 다룬다
- 서로 다른 절차가 섞이기 시작하면 별도 runbook으로 분리한다
