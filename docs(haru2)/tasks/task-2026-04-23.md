## 계획

현재 상태 요약: 최신 기준 상태는 [status-2026-04-23.md](../status/status-2026-04-23.md)에 있고, 오늘 작업은 `infra/docs`를 기준으로 루트 `docs(haru2)`의 허브 문서와 reference 축을 애플리케이션 작업 레이어에 맞게 다시 정리하는 데 집중한다.

1. `infra/docs`와 루트 `docs(haru2)`의 허브 문서 차이를 정리한다
   - `README.md`, `status/README.md`, `tasks/README.md`, `runbooks/README.md`, `troubleshooting/README.md`, `architecture.md`를 비교한다
   - 그대로 가져올 공통 골격과 앱 레이어용으로 바꿔야 할 차이를 나눈다
2. 루트 `reference/` 축을 보강한다
   - `application-workflow-from-idea-to-operation.md`, `application-source-of-truth.md`, `operational-notes.md`를 추가한다
   - `reference/README.md`, `document-writing-rules.md`, 기존 reference 문서의 링크와 경로 규칙을 현재 canonical 경로 기준으로 맞춘다
3. 오늘 반영한 범위와 남은 후속 작업을 기록한다
   - 최신 status/task 기준이 오늘 작업 축을 설명할 수 있게 정리한다
   - 아직 남은 옛 경로 표기나 후속 문서 개선은 `보류 / 다음 액션`에 남긴다

## 진행

- 시작 전: 루트 `AGENTS.md`와 `.codex/skills/`는 `docs(haru2)` 기준으로 정렬됐지만, `docs(haru2)` 허브 문서와 reference 축은 아직 `infra/docs`보다 얇았다
- `infra/docs`의 `README`, `reference/README`, `runbooks/README`, `troubleshooting/README`, `infra-workflow-from-idea-to-operation.md`, `deployment-source-of-truth.md`, `operational-notes.md`를 기준본으로 다시 읽었다
- 루트 `docs(haru2)` 문서를 아래 방향으로 정리했다
  - `README.md`, `architecture.md`, `status/README.md`, `tasks/README.md`, `runbooks/README.md`, `troubleshooting/README.md`를 앱 레이어 기준으로 보강했다
  - `reference/README.md`와 `document-writing-rules.md`를 현재 canonical 경로와 새 reference 축에 맞게 갱신했다
  - `application-workflow-from-idea-to-operation.md`, `application-source-of-truth.md`, `operational-notes.md`를 새로 추가했다
  - `application-dev-vs-prod.md`, `application-code-change-summary.md`의 링크와 경로 표기를 현재 문서 규칙에 맞게 최소 보정했다
- 오늘 기준 최신 루트 status/task도 추가해, 현재 작업 축이 오래된 3월 문서에만 머물지 않도록 시작 기준을 다시 세웠다
- 애플리케이션 레이어 특성에 맞춰 workflow 보강도 진행했다
  - `web/` 변경 시 관련 `web/docs/` 확인을 필수 단계로 올렸다
  - `ai/` 변경 시 관련 `ai/docs/`와 필요한 `ai/README.md` 확인을 필수 단계로 올렸다
  - 루트, `ai/`, `web/`의 Claude 전용 설정 디렉터리와 ignore 파일을 제거하고, 활성 문서 참조를 `ai/README.md` 기준으로 다시 맞췄다
  - `pnpm check-all`, `uv run make lint`를 최소 실행 검증 기준으로 문서와 `evaluate-implementation` skill에 연결했다
  - 루트 `reference/`보다 관련 `web/docs/`, `ai/docs/`를 우선하는 source of truth 규칙을 명시했다
- 원 개발자 쪽 Claude 자산 중 현재 스택과 직접 맞는 기술 자산을 Codex 스킬로 이식했다
  - `Gemini prompting`, `Prisma v7`, `LangGraph stream`, `web server architecture`, `web performance patterns`를 새 `.codex/skills/`로 추가했다
  - 예전 Claude 전용 자산의 장점은 유지하되, 현재 루트 문서/검증 흐름과 맞는 Codex 스타일 설명으로 다시 썼다
  - `evaluate-implementation`에 남아 있던 `ai/CLAUDE.md` 참조를 `ai/README.md`로 정리했다
